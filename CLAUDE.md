## Project Overview

PDF-to-CSV converter that extracts tables from vector PDF files using [Camelot](https://camelot-py.readthedocs.io/). Supports CSV, JSON, and Excel output. Optionally applies business rules (règles métiers) after extraction.

## Setup

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
# Optionnel : installer le package en mode éditable pour éviter les manipulations de sys.path
pip install -e .
```

## Running the Converter

The entry point is `bin/run_converter.py`. It can be run from any directory:

```powershell
python bin\run_converter.py --input-dir assets\sample_base_first --output-dir tmp --output-format csv --verbose
```

Key flags: `--output-format` (csv/json/excel), `--verbose`, `--skip-existing`, `--check-rules`.

Output lands in a timestamped subdirectory: `<output-dir>/output_YYYY-MM-DD_HH-MM-SS/`.

## Running Tests

Tests can be run from the project root:

```powershell
pytest tests/ -v
pytest tests/test_converter.py::test_convert_pdf_file_with_rules -v
```

## Architecture

Le code suit le pattern MVC :

```
bin/run_converter.py                        # Point d'entrée CLI
src/documents_utils/
    models/
        pdf_reader.py                       # extract_tables() — wraps Camelot
        rules.py                            # check_regle_1() — logique métier pure
    views/
        cli_view.py                         # Tous les print/affichages
    controllers/
        converter.py                        # convert_file(), convert_directory(), main()
    pdf_to_csv.py                           # Shim de compatibilité → controllers/converter
    rules.py                                # Shim de compatibilité → models/rules
tests/
    test_converter.py                       # pytest; assets dans tests/assets/
assets/                                     # PDFs exemples pour tests manuels
```

**Data flow:** `run_converter.py` → `converter.convert_directory()` itère les PDFs → `convert_file()` appelle `pdf_reader.extract_tables()` → écrit les fichiers → appelle optionnellement `rules.check_regle_1()` → `cli_view` affiche les résultats.

**Camelot flavor:** `lattice` is tried first (requires grid lines). If it returns 0 tables, `stream` is tried automatically as fallback. Scanned/image PDFs produce zero tables with both flavors.

**Business rules** (`rules.py`): Each rule reads an extracted CSV and validates domain-specific conditions. New rules should follow the same pattern: accept a CSV path, return bool, print results. `--check-rules` is silently ignored (with a warning) when `--output-format` is not `csv`.

- **Règle 1 (`check_regle_1`)** : le montant *"total saisissable"* doit être strictement inférieur au montant *"solde bancaire insaisissable à retenir"*. Contexte : règle juridique bancaire — le montant saisissable par un créancier ne peut pas dépasser le seuil légal insaisissable (minimum vital protégé par la loi). La fonction cherche ces libellés dans la première colonne du CSV et compare les montants de la deuxième colonne (format `"0.59 EUR"`).

**PermissionError suppression:** A custom `sys.unraisablehook` in `pdf_to_csv.py` silences `PermissionError` on Windows temp files generated by Camelot/pdfminer during garbage collection.
